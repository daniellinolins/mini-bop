# Mini BOP Academy — Integration Checklist

> Checklist para integração e validação da documentação antes de publicar uma nova versão.

---

# Objetivo

Este documento serve como um guia de revisão para garantir que a Academy permaneça consistente, navegável e alinhada ao código do projeto.

---

# Estrutura

## README principal

- [ ] Possui link para a Academy.
- [ ] Possui descrição da Academy.
- [ ] Possui links para os idiomas disponíveis.

---

## Home da Academy

- [ ] Explica o propósito da Academy.
- [ ] Possui Learning Path.
- [ ] Possui links para PT-BR, EN-US e FR-FR.
- [ ] Possui tempo estimado de estudo.

---

## Módulos

Cada capítulo deve possuir:

- [ ] Objetivo.
- [ ] Resumo.
- [ ] Link para capítulo anterior.
- [ ] Link para próximo capítulo.
- [ ] Link para Home da Academy.
- [ ] Referências relacionadas.

---

## Conteúdo Técnico

Verificar se:

- [ ] Está baseado no código do Mini BOP.
- [ ] Não apresenta funcionalidades inexistentes.
- [ ] Diferencia implementação atual de Roadmap.
- [ ] Diferencia implementação atual de Technical Debt.

---

## Diagramas

- [ ] Mermaid renderiza corretamente.
- [ ] Fluxos representam o código atual.
- [ ] Diagramas possuem títulos claros.

---

## Navegação

- [ ] Todos os links funcionam.
- [ ] Não existem links quebrados.
- [ ] A sequência dos módulos está correta.

---

## Idiomas

- [ ] PT-BR atualizado.
- [ ] EN-US sincronizado.
- [ ] FR-FR sincronizado.

---

## Publicação

Antes de criar uma nova release:

- [ ] Revisão ortográfica.
- [ ] Revisão técnica.
- [ ] Revisão de links.
- [ ] Revisão dos diagramas.
- [ ] Atualização do CHANGELOG (quando aplicável).

---

# Critério de aprovação

A documentação somente deve ser publicada quando todas as verificações acima estiverem concluídas.
