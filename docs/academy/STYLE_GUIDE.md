# Mini BOP Academy — STYLE_GUIDE

> Guia visual e editorial para manter uma identidade consistente em toda a Academy.

---

# Objetivo

Este documento define convenções visuais e editoriais para todos os capítulos da Mini BOP Academy.

---

# Estrutura recomendada

Todos os módulos devem seguir, sempre que possível, esta sequência:

1. Objetivo
2. Pré-requisitos
3. Contexto de Negócio
4. Conceitos Fundamentais
5. Implementação no Mini BOP
6. Diagrama Mermaid
7. Decisões de Engenharia
8. Resumo
9. Próximo módulo

---

# Ícones

| Ícone | Utilização |
|-------|------------|
| 📘 | Capítulo |
| 🎯 | Objetivo |
| 💼 | Contexto de Negócio |
| ⚙️ | Implementação |
| 🏗️ | Arquitetura |
| 💡 | Observação importante |
| ⚠️ | Atenção |
| 📊 | Analytics |
| 🚀 | Evolução futura |

---

# Diagramas

Utilize Mermaid para representar:

- Fluxos
- Pipelines
- Dependências
- Arquiteturas
- Sequências

Evite diagramas excessivamente complexos.

---

# Código

Quando apresentar código:

- incluir apenas trechos relevantes;
- explicar o propósito do exemplo;
- manter consistência com o código do projeto.

---

# Terminologia

Utilize sempre os termos definidos em `GLOSSARY.md`.

Evite criar sinônimos para o mesmo conceito.

---

# Navegação

Cada capítulo deve conter:

- link para a Home da Academy;
- capítulo anterior;
- capítulo seguinte;
- documentação relacionada.

---

# Idiomas

A estrutura deve permanecer idêntica entre:

- PT-BR
- EN-US
- FR-FR

Mudanças estruturais devem ser aplicadas nos três idiomas.

---

# Objetivo final

A Academy deve transmitir a sensação de um único livro, e não de documentos independentes.
