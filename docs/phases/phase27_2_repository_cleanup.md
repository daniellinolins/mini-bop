# Phase 27.2 — Repository Cleanup Baseline

This cleanup prepares Mini BOP for the final documentation and demo kit work.

## Scope

The repository remains a local Enterprise Data Engineering reference implementation. Docker artifacts are intentionally excluded from the current delivery baseline.

## Runtime artifacts to remove from version control

The following files and folders are generated by local tools and should not be committed:

```text
metastore_db/
.metastore_db/
derby.log
spark-warehouse/
warehouse/
__pycache__/
*.pyc
*.tmp
*.bak
*.old
*.orig
```

## Cleanup scripts

Two equivalent cleanup scripts are provided:

```bash
bash scripts/platform/264_cleanup_local_runtime_artifacts.sh
```

```powershell
powershell -ExecutionPolicy Bypass -File scripts/platform/264_cleanup_local_runtime_artifacts.ps1
```

Both scripts assume execution from the project root or use `MINI_BOP_PROJECT_ROOT` / `-ProjectRoot` to resolve the repository location.

## Git ignore updates

`.gitignore` was updated to exclude local runtime artifacts, Python caches, Derby/Hive metastore files, Spark warehouse folders, local logs, temporary files and packaged artifacts.

## Documentation language baseline

The project documentation uses an engineering delivery tone. Terms such as "learning project", "educational implementation" and "lab" are avoided in favor of:

- reference implementation
- Enterprise Data Engineering platform
- operational capability
- engineering practice
- platform readiness
